# Plan: Migrate Navbar to ShadCN NavigationMenu

## Context

The site header currently uses hand-styled `<Link>` elements for navigation. We want to adopt ShadCN NavigationMenu components for a more polished, consistent look with built-in hover/focus states. This is a **visual uplift only** — same links, no dropdowns, no new items. Both desktop and mobile nav will be updated.

## Steps

### 1. Install ShadCN NavigationMenu

```bash
npx shadcn@latest add navigation-menu -y
```

Creates `components/ui/navigation-menu.tsx` with all sub-components and `navigationMenuTriggerStyle()`.

### 2. Rewrite `components/nav-links.tsx`

Replace bare `<Link>` elements with NavigationMenu components:

```tsx
<NavigationMenu>
  <NavigationMenuList>
    <NavigationMenuItem>
      <NavigationMenuLink asChild className={cn(navigationMenuTriggerStyle(), activeClass)}>
        <Link href="/">Home</Link>
      </NavigationMenuLink>
    </NavigationMenuItem>
    ...
  </NavigationMenuList>
</NavigationMenu>
```

- Active state: `cn(navigationMenuTriggerStyle(), pathname === href && "bg-accent text-accent-foreground font-medium")`
- Inactive links get the trigger style's built-in hover/focus states automatically
- Stays a client component (`"use client"`) with same export name — no changes needed in `navbar.tsx` imports

### 3. Update `components/basket-badge.tsx`

Apply `navigationMenuTriggerStyle()` to the basket link for visual consistency. Keep it **outside** NavigationMenu (it has the Badge overlay and useBasket hook):

- Replace manual `text-sm transition-colors hover:text-foreground` / `text-muted-foreground` classes with `cn(navigationMenuTriggerStyle(), ...activeClass)`
- Badge count display unchanged

### 4. Adjust spacing in `components/navbar.tsx`

The `<nav>` element currently has `gap-6`. Since NavigationMenu items have built-in padding from `navigationMenuTriggerStyle()`, reduce to `gap-2` (or adjust visually as needed). BasketBadge also gets trigger-style padding so the gap between items can be tighter.

### 5. Restyle mobile Sheet links in `components/mobile-nav.tsx`

Use `navigationMenuTriggerStyle()` (just the style function, not the full component) for the nav links inside the Sheet:

- Replace manual link classes with `cn(navigationMenuTriggerStyle(), "w-full justify-start", ...activeClass)`
- `w-full justify-start` overrides ensure links stretch full-width and left-align in the vertical Sheet layout
- Auth section (email, Sign out, Sign in) stays as-is — visually secondary, separated by border

### 6. Update tests

**`components/__tests__/nav-links.test.tsx`:**
- Active state assertions change from `toHaveClass("font-medium")` to checking for `"bg-accent"` (or keep `font-medium` if we include it in the active class)
- Link rendering tests (`getByRole("link")`) should still work since `NavigationMenuLink asChild` renders the child `<Link>` → `<a>`
- May need to mock `@/components/ui/navigation-menu` if Radix primitives cause jsdom issues (try without first)

**`components/__tests__/basket-badge.test.tsx`:**
- No assertion changes needed — tests check link presence and badge count, not styling classes

### 7. Verify

```bash
npm run typecheck    # No TS errors from new imports
npm test             # All tests pass
npm run lint         # No ESLint violations
npm run dev          # Visual check: desktop nav, mobile sheet, active states, hover/focus
```
